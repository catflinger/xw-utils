Readme

Browser compatibility

So far I have run the application successfully with Chrome, Firefox, Opera, Edge and Safari (iOS) browsers. I have also used the application successfully on a mobile phone running Android with Chrome.

The application will function on Internet Explorer 11 on Windows but has display issues where the text and images overlap on some screen sizes.

Known issues
The clue editor misinterprets letter counts like (6, 2 words) as _ _ _ _ _ _ / _ _

The blogger page should not display "answer does not fit space available message" because the answer is a mismatch for the grid.
Warning relating to the grid should only be displayed in the solver page.

In the solver pressing ESC should cancel any highlighting

When failing to download a puzzle distinguish between "not found" and general errors.  For example: when trying
to open future puzzles in the Independent they may not be there yet.  This is not an error, so show a more appropriate 
error message.

Sometimes the app does not correctly distinguish between a password mismatch and a network error, both show same message.

Google diary shows wrong error message. If the user is not loggin in "Failed to get diary" is shown instead of "Please log-in and try again".

Find out why AZED 2482 did not load the grid properly, some bars were missing.
